Three Approaches for the EKG Drill

Same core idea, different UX philosophies. Click to select.

A

Flashcard Flip

Full-screen EKG strip with a two-button choice: MMVT or PMVT. Tap to answer, card flips to reveal the diagnosis with 3-4 bullet points explaining the key features. Swipe/click for next card.

Think: Anki meets ECG. Minimal UI. Fast reps. Spaced repetition runs silently underneath.

Pros

  • Fastest drill cycle
  • Zero cognitive overhead
  • Mobile-friendly

Cons

  • Binary choice could feel too easy
  • No room for nuance until the reveal
B

Feature Spotter

EKG strip shown with clickable annotation zones. User identifies key features first (e.g., "QRS morphology varies beat-to-beat," "regular R-R intervals"), then makes the MMVT/PMVT call. Feedback highlights which features they caught and which they missed.

Think: active reading exercise. Forces you to look before you label.

Pros

  • Teaches the "why," not just the "what"
  • Builds systematic reading habits
  • More engaging per card

Cons

  • Slower per rep
  • More complex to build
  • Annotation zones need careful mapping per image
C

Progressive Reveal

Strip shown. User picks MMVT or PMVT. Then a follow-up: "What's the single most important feature that told you?" with 3-4 choices (e.g., "changing QRS axis," "uniform QRS morphology," "sinusoidal amplitude variation"). Two-step answer, richer feedback.

Think: halfway between A and B. Fast but forces articulation.

Pros

  • Quick but deeper than pure binary
  • Tests reasoning, not just pattern matching
  • Simple to build (no image annotations)

Cons

  • Second question could feel tedious after many reps
  • Feature choices need to be well-curated per strip

My recommendation: C (Progressive Reveal). It hits the sweet spot: fast enough for spaced repetition volume, but the "why" question separates real understanding from lucky guessing. And it's straightforward to build with static images.
